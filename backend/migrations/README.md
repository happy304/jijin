# Database migrations

Alembic manages every schema change for the Fund Quant Platform backend.

## Quick start

```bash
# From backend/
alembic upgrade head          # apply all pending migrations
alembic revision -m "add_x"   # author a new empty migration
alembic revision --autogenerate -m "sync models"
alembic downgrade -1          # revert the last migration
```

## Configuration

* `alembic.ini` at `backend/alembic.ini` holds tooling configuration.
* The database URL is **not** stored in the ini file. `env.py` loads it
  from `app.core.config.Settings.database_sync_url`, which in turn reads
  `DATABASE_SYNC_URL` from the environment (or `.env`).
* Override for one-off runs: `alembic -x url="postgresql+psycopg://..." upgrade head`.

## Target metadata

`env.py` imports `app.data.models.Base`, so autogenerate will diff the
live database against every ORM class registered under that base.
Adding a new model file automatically makes it visible to Alembic —
just ensure the file is imported somewhere in `app.data.models`.

## Naming

Constraint names follow a deterministic convention configured on
`Base.metadata` (see `app/data/models/__init__.py`). This keeps
autogenerated migrations stable across contributors.
